How it works:

1. The Sender hashes their Message with the Secret Key to generate a MAC (HMAC-SHA256).

2. They send the cleartext Message along with the MAC over the network.

3. The Receiver takes the incoming Message and applies their own copy of the Secret Key to calculate the Expected MAC.

4. If the Expected MAC matches the Received MAC, the message is Authentic (sent by someone with the key) and Intact (has not been altered). Try changing 1 letter in the Received Message to see verification fail!
